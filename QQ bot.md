
# 一、整体架构（你未来项目的“骨架”）

先给你一个**清晰、可扩展的结构**（你后面所有功能都往这里塞）：

```text
qq-bot/
├── main.go                # 程序入口
├── config/               # 配置（端口、token等）
├── bot/                  # 核心逻辑
│   ├── client.go         # WebSocket连接QQ
│   ├── handler.go        # 消息入口（总调度）
│   ├── context.go        # 封装消息上下文
│
├── command/              # 命令系统（/xxx）
│   ├── router.go         # 命令分发
│   ├── ping.go           # 示例命令
│
├── event/                # 事件系统（入群/退群等）
│   ├── group.go
│
├── service/              # 业务逻辑（核心能力）
│   ├── user.go           # 用户信息
│   ├── repeat.go         # 连续发言检测
│   ├── course.go         # 课表系统
│
├── storage/              # 数据存储
│   ├── db.go             # SQLite连接
│
├── image/                # 图片生成（重点模块🔥）
│   ├── tom.go            # 汤姆嘲笑
│   ├── rank.go           # 排行榜
│
└── utils/
    ├── http.go           # 请求工具（下载头像）
```

# 二、系统运行流程（你必须理解这个）

以后所有功能，都是走这个链路：

```text
QQ群消息
   ↓
NapCat / OneBot
   ↓（WebSocket）
你的 bot/client.go
   ↓
handler.go（统一入口）
   ↓
判断类型：
   ├── 是命令 → command/router.go
   ├── 是事件 → event/*
   ├── 是普通消息 → service/*
   ↓
处理完
   ↓
发送消息回QQ群
```

👉 这就是一个**事件驱动架构（非常重要）**

# 三、核心设计思想（你必须掌握）

我帮你提炼成3个关键词：

---

## 1️⃣ “入口统一”

所有消息都从一个地方进：

```go
func HandleMessage(msg Message) {
    // 所有逻辑从这里分发
}
```

👉 好处：

- 不乱
    
- 好维护
    

---

## 2️⃣ “命令系统”

比如：

```text
/ping
/今日小猪
/课表
```

设计成：

```go
map[string]Handler
```

👉 你以后加功能就是：

👉 新建一个文件 + 注册

---

## 3️⃣ “服务层（Service）”

比如：

- repeat.go（检测复读）
    
- course.go（课表）
    

👉 这些是“能力模块”
