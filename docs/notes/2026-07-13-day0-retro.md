## 过程中遇到的问题

### 1. NapCat 不是 npm 包
- **错误假设**: `npm install -g napcat`
- **实际**: NapCat 是 GitHub Release zip + AUR 包 `napcat-qq`
- **教训**: 在推荐安装方式之前先验证

### 2. OneBot WebSocket 默认不开启
- 安装后 `websocketServers: []` 为空
- 需要手动写入配置，且必须在 NapCat **启动前**写入
- 之前多次尝试失败的根本原因：先启动了 NapCat，后改配置，NapCat 读的是旧配置

### 3. NapCat 登录后才能开启 OneBot 服务
- NapCat 完成 QQ 扫码登录后约 15 秒才开启 OneBot WebSocket
- screen 后台启动时无法扫码，必须前台运行
- 之前多次 "WS:3001 MISSING" 要么是因为运行了旧配置的实例，要么是因为没登录

### 4. 多实例问题
- `screen -dmS napcat napcat-qq` 执行了多次，创建了 3 个同时运行的实例
- 多个 QQ 进程争抢同一账号导致异常
- screen 后台运行不方便调试
